## 1. 접속

- [ ] https://tronscan.org/#/contracts/verify 접속

---

## 2. 계약 정보 (위쪽)

| 기입란 | 입력값 | 비고 |
|--------|--------|------|
| **Contract Address** (계약 주소) | 배포한 컨트랙트 주소 (T로 시작) | 예: TNXDJXHKBXZFWLksXpvccy9oKZNw9kHPVS |
| **Solidity Compiler Version** | 배포 시 사용한 버전 | **tron_v0.8.6** (TronIDE 기본) |
| **Optimization** (최적화) | **No** (비활성화) | TronIDE 기본이면 No. 실패 시 Activated 시도 |
| **Runs** | **0** | Optimization No면 0 |
| **License** | None 또는 MIT | |
| **VM version?** | **default** | TronIDE 기본 |
| **ViaIR?** | **No** | IR 컴파일 사용 여부. TronIDE 기본은 No |

→ **기입란 전부·역할:** `소스검증_기입란_전부_역할.md`

---

## 3. Main Contract (수요 계약)

| 배포한 Contract Name | 입력값 |
|----------------------|--------|
| **LinkUSD** (재배포 시) | `LinkUSD` |
| **TetherUSD** (현재 메인넷) | `TetherUSD` |

→ 배포에 쓴 .sol 파일의 `contract 이름`과 **정확히 일치**해야 함.

---

## 4. 계약 파일 업로드 ⭐ (가장 중요)

- [ ] **"계약서 파일을 업로드하세요"** 또는 **"Upload Contract File(s)"** 클릭
- [ ] 배포에 사용한 .sol 파일 선택 후 업로드

| 배포한 Contract | 업로드할 파일 |
|-----------------|---------------|
| **LinkUSD** | `LinkUSD_TRON.sol` |
| **TetherUSD** | `TetherUSD_Mainnet_Verify.sol` |

→ **파일 안 올리면 검증 불가.** "계약 코드가 확인되지 않았습니다" 계속 뜸.

---

## 5. Verify And Publish

- [ ] **Verify And Publish** (또는 **검증 및 게시**) 클릭
- [ ] 실패 시: `Verify_실패_해결.md` 참고 (Optimization, Runs, Compiler 바꿔서 재시도)

---

## 6. 체크 (제출 전)

- [ ] Contract Address 입력
- [ ] Main Contract = 배포한 contract 이름 (LinkUSD 또는 TetherUSD)
- [ ] Optimization, Runs, Compiler, **VM Version** 맞춤
- [ ] **.sol 파일 업로드** (빼먹지 말 것)
- [ ] Verify And Publish 클릭
