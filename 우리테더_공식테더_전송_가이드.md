# 우리 테더 → 상대 지갑 입금 (모든 방법 정리)

## 1. 3번(브릿지 컨트랙트)이란?

**스마트 컨트랙트** 하나를 두고:
- 사용자가 **우리 USDT**를 컨트랙트 주소로 전송 + 수신자 주소 입력
- 컨트랙트가 **공식 USDT**를 수신자에게 1:1로 전송

**조건:** 컨트랙트에 공식 USDT를 미리 넣어둬야 함. (프로젝트가 공식 USDT 보유 필요)

---

## 2. TronLink / MetaMask / Trust Wallet 비교

| 지갑 | TRON 지원 | 커스텀 토큰 자동 표시 |
|------|-----------|------------------------|
| **TronLink** | ✅ | ❌ 수동 추가만 가능 |
| **MetaMask** | ✅ (TRON 지원) | ❌ 화이트리스트 토큰 위주, 커스텀은 수동 |
| **Trust Wallet** | ✅ | ❌ 수동 추가 (Manage Crypto) |

**공통:** TRON 커스텀 토큰은 어느 지갑이든 **직접 추가**해야 보임.

---

## 3. 이더리움·폴리곤 방식은 다른가?

**아니다.** ERC20(이더리움), Polygon 토큰도 **커스텀 토큰은 수동 추가**가 필요함.

- MetaMask: Token Auto-Detection이 있지만 **주로 유명 토큰**만 자동 표시
- 알 수 없는 커스텀 토큰 → 여전히 수동 추가

**TRC20(TRON)과 ERC20(이더리움) 메커니즘은 같음.** 커스텀 토큰은 지갑 기본 목록에 없으면 추가해야 함.

---

## 4. 토큰 추가 없이 입금되는 유일한 방법

**공식 USDT**를 보내는 것. 공식 USDT는 대부분 지갑에 기본으로 들어 있음.

| 보내는 토큰 | 수신자 | 토큰 추가 필요 |
|-------------|--------|----------------|
| 우리 USDT | TronLink, MetaMask, Trust Wallet | ✅ 필요 |
| 공식 USDT | TronLink, MetaMask, Trust Wallet | ❌ 대부분 불필요 |

---

## 5. 수신자 부담 줄이기: 원클릭 토큰 추가

우리 USDT를 받은 사람이 **우리 페이지**에 들어와서 "지갑에 토큰 추가" 버튼을 한 번만 누르면 됨. (주소 복붙 대신)

- usdt-sunpump.html → **"TronLink에 토큰 추가"** 버튼
- TronLink, MetaMask(TRON) 등 `wallet_watchAsset` 지원 지갑에서 동작

---

## 6. 방법 정리

| 방법 | 설명 |
|------|------|
| **A. 공식 USDT 전송** | 공식 USDT 보유 시 → 수신자 주소로 전송. 토큰 추가 불필요 |
| **B. 풀 + 스왑** | 우리/공식 USDT 풀 생성 → 스왑 후 공식 USDT 전송 |
| **C. 브릿지 컨트랙트** | 우리 USDT → 컨트랙트, 컨트랙트 → 수신자에게 공식 USDT. 공식 USDT 예치 필요 |
| **D. 원클릭 추가** | 우리 USDT 전송 후, 수신자가 우리 페이지에서 "토큰 추가" 한 번 클릭 |

---

## 7. 결론

- **토큰 추가 없이** 입금되게 하려면 → **공식 USDT**만 보내면 됨.
- 우리 USDT를 보내면 → 수신자가 토큰을 추가해야 보임. (TronLink, MetaMask, Trust Wallet 모두 동일)
- **이더리움·폴리곤도** 커스텀 토큰은 같은 방식. 자동 표시는 유명 토큰 위주.
