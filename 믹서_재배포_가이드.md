## 2. 설계 선택

### 선택 A: 단순 재배포 (USDT TeatherUSD 전용 믹서)

- **목적**: USDT TeatherUSD(decimals 18)만 믹싱
- **방법**: `TokenMixer.sol` 수정 → FIXED_AMOUNT = `1000 * 10^18` → 새로 배포
- **constructor**: `_token` = `TGacvY1cTSLKgWEPxXqWt8FFAQ86vpzM1U`
- **장점**: 코드 변경 최소, 바로 배포 가능
- **단점**: 공식 USDT는 기존 믹서 사용 (믹서 2개 운영)

### 선택 B: 재설계 (토큰별 금액 지원)

- **목적**: 공식 USDT + USDT TeatherUSD 한 믹서에서 둘 다 지원
- **방법**: MultiTokenMixer 수정 → 토큰별 amount 저장 → 새로 배포
- **장점**: 믹서 1개로 통합
- **단점**: 컨트랙트 수정·테스트 필요

### 선택 C: 그대로 유지

- **공식 USDT**: 현재 믹서 그대로 사용
- **USDT TeatherUSD**: 믹서 미지원 (또는 선택 A로 별도 믹서 배포)

## 3. 재배포 절차 (선택 A 기준)

### 3-1. TokenMixer.sol 수정

```solidity
// 변경 전
uint256 public constant FIXED_AMOUNT = 1000 * 10**6;

// 변경 후 (decimals 18)
uint256 public constant FIXED_AMOUNT = 1000 * 10**18;
```

### 3-2. TronIDE에서 배포

1. https://www.tronide.io 접속 → **Mainnet**
2. **+** → `TokenMixer.sol` 생성 → 수정된 코드 붙여넣기
3. **Compile** 클릭
4. **Deploy** 탭 → CONTRACT: `TokenMixer`
5. **constructor** `_token`: `TGacvY1cTSLKgWEPxXqWt8FFAQ86vpzM1U`
6. **Deploy** → TronLink **Confirm**

### 3-3. 배포 후

- 새 믹서 주소 확인
- `믹서_사용_가이드.md`에 주소·approve 금액(`1000 * 10^18`) 반영
