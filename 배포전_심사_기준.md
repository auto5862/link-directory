## 2. 배포전 심사 기준치 (전부 통과해야 배포)

### 2-1. 금지 항목 (하나라도 있으면 ❌ → 정정 후 재심사)

| # | 기준 | 통과 |
|---|------|------|
| 1 | **Contract Name** ≠ TetherUSD, Tether, USDT | ☐ |
| 2 | **name** ≠ "Tether USD", "Test USD", "USDT", "TUSD", "TrueUSD" | ☐ |
| 3 | **symbol** ≠ "USDT", "TUSD" | ☐ |
| 4 | 코드 안에 **Tether, USDT, TrueUSD, TUSD** 단어 없음 | ☐ |

### 2-2. 필수 항목 (전부 있어야 ✓)

| # | 기준 | 통과 |
|---|------|------|
| 5 | `contract [이름]` — 고유한 이름 | ☐ |
| 6 | `name`, `symbol` — 배포 후 변경 불가, 최종 확인 | ☐ |
| 7 | `transfer`, `approve`, `transferFrom` 기본 함수 있음 | ☐ |
| 8 | `pragma solidity` 버전 명시 | ☐ |

### 2-3. 선택 (있으면 좋음)

| # | 기준 | 통과 |
|---|------|------|
| 9 | burn, lock 등 추가 기능 | ☐ |
| 10 | owner, onlyOwner 등 권한 관리 | ☐ |

## 3. 심사 절차

| 순서 | 할 일 |
|------|-------|
| 1 | 각 .sol 문서 열기 |
| 2 | 위 기준치 2-1, 2-2 전부 체크 |
| 3 | **기준치 미달** → 코드 수정 → 다시 1번부터 |
| 4 | **전부 통과** → 배포 진행 |

---

## 4. 기준치 미달 시 정정 예시

| 미달 | 정정 |
|------|------|
| contract TetherUSD | → contract LinkUSD (또는 다른 고유 이름) |
| name = "Tether USD" | → name = "Link USD" (또는 다른 이름) |
| symbol = "USDT" | → symbol = "LUSD" (또는 다른 심볼) |
| 코드에 Tether 단어 | → 전부 제거 또는 교체 |
