# 소스 검증 — 기입란 전부 + 각자 하는 일

**"verification failed" = 코드 잘못이 아님. 파라미터(컴파일 설정)가 배포 시와 안 맞는 것.**

---

## ⚠️ 검증 실패 ≠ 코드 오류

| 오해 | 사실 |
|------|------|
| "코드가 잘못됐나?" | **아님.** |
| "verification failed" 의미 | **Optimization, Runs, Compiler, VM Version, evmVersion** 등이 **배포 시 사용한 값과 다름** |

→ 코드는 맞아도, 위 파라미터 하나라도 다르면 **바이트코드가 안 맞아서** 실패함.

---

## 기입란 전부 + 각자 하는 일 (화면 순서)

| 기입란 | 하는 일 | 입력 예시 |
|--------|---------|-----------|
| **Contract Address** | 검증할 컨트랙트 주소 | TNXDJXHKBXZFWLksXpvccy9oKZNw9kHPVS |
| **Solidity Compiler Version** | 배포 시 사용한 Solidity 컴파일러 버전 | tron_v0.8.25+commit.77bd169f |
| **Optimization?** | 배포 시 최적화 사용 여부 | Not Activated 또는 Activated |
| **ViaIR?** | IR(중간 표현) 컴파일 사용 여부 | **No** (TronIDE 기본) |
| **Main Contract** | `contract` 뒤의 이름 | TetherUSD 또는 LinkUSD |
| **License?** | 소스 라이선스 | MIT License(MIT) 또는 None |
| **Runs?** | 최적화 사용 시 runs 수 (No면 0) | 0 또는 200 |
| **VM version?** | TRON VM 버전 (배포 시 설정) | default |
| **Upload Contract File(s)** | 배포에 쓴 .sol 파일 | TetherUSD.sol 또는 LinkUSD_TRON.sol |

---

## ViaIR? vs VM version?

| 항목 | 역할 |
|------|------|
| **ViaIR?** | Solidity IR(중간 표현) 컴파일 사용 여부. TronIDE 기본: **No** |
| **VM version?** | TRON 가상머신(TVM) 버전. TronIDE 기본: **default** |

→ ViaIR = Yes면 컴파일 방식이 달라짐. 실패 시 **ViaIR: Yes** 시도 가능.

---

## 시도 순서 (전부 기입 후)

### 1차 (현재 화면 그대로)
| 항목 | 값 |
|------|-----|
| Optimization? | **Not Activated** |
| ViaIR? | **No** |
| Runs? | **0** |
| VM version? | **default** |
| Compiler | tron_v0.8.25 |
| Main Contract | TetherUSD |
| License? | MIT License(MIT) |
| 업로드 | TetherUSD.sol |

### 2차 (1차 실패 시)
| 항목 | 값 |
|------|-----|
| ViaIR? | **Yes** |
| 나머지 | 1차와 동일 |

### 3차 (2차 실패 시)
| 항목 | 값 |
|------|-----|
| Optimization? | **Activated** |
| Runs? | **200** |
| ViaIR? | No |
| 나머지 | 1차와 동일 |
