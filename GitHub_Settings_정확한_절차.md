# GitHub Settings — 정확한 절차 (혼동 방지)

---

## Settings 들어왔을 때

**지금 보이는 화면:** `일반적인` (General) — 여기서 할 일 거의 없음.

---

## 해야 할 것: Pages 활성화

1. Settings 페이지 **왼쪽 메뉴** 보기
2. **General 아님** — 아래로 스크롤
3. **"페이지" (Pages)** 클릭
   - 위치: "코드 및 자동화" 또는 "코드스페이스" 근처
4. **Source** → **Deploy from a branch** 선택
5. **Branch**: main
6. **Folder**: / (root)
7. **Save** 클릭

→ `https://auto5862.github.io/link-directory/` 생김

---

## 코드공개 (Code visibility)

| 구분 | 상태 |
|------|------|
| **저장소가 Public** | 코드 이미 공개됨. 누구나 볼 수 있음 |
| **Private** | 본인만 봄 |

저장소 이름 옆에 "공공의" (Public) 뜨면 → **코드공개 완료**.

## 지금까지 한 것 vs 안 한 것

### ✅ 한 것
- 저장소 생성 (link-directory)
- Description 입력
- README 업로드
- LICENSE (MIT)

### ❌ 아직 안 한 것
- **Pages 활성화** (Settings → 왼쪽 **Pages** 클릭)
- 나머지 파일 업로드 (LinkUSD_TRON.sol, 가이드, HTML 등) — 안 올렸으면 Add file → Upload files
- 메인 지갑 검증
- TronIDE 배포
- TronScan Record Token, Verify Contract 등

## "다 한 거니?" — 아니다

**GitHub 쪽:** 저장소 + README + Pages 활성화 + 파일 업로드
