## 3. 핵심 질문 — 메인넷 컨트랙트에 burn, lock 있나?

TronScan에서 해당 컨트랙트 페이지 → **Contract** 탭 → **Read/Write Contract** 들어가서:

| 함수 | 있으면 | 없으면 |
|------|--------|--------|
| **burn** | 배포된 건 LinkUSD_TRON.sol 계열 (기능 추가된 버전) | 배포된 건 TetherUSD.sol (기본 버전) |
| **lock** | 위와 동일 | 위와 동일 |

---

## 4. 검증 실패 가능 원인

| 원인 | 설명 |
|------|------|
| **파일 불일치** | burn/lock 있는데 TetherUSD.sol 업로드 → 바이트코드 다름 → 실패 |
| **파일 불일치** | burn/lock 없는데 LinkUSD_TRON.sol 업로드 → 바이트코드 다름 → 실패 |
| **배포 후 수정** | 배포 후 .sol 파일을 수정했으면 → 바이트코드 다름 → 실패 |
| **컴파일 옵션** | Optimization, ViaIR, Compiler, VM version이 배포 시와 다름 → 실패 |
| **테스트넷 vs 메인넷** | 테스트넷에서 쓴 설정과 메인넷 배포 시 설정이 다를 수 있음 |
