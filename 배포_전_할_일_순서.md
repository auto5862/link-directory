## 중요: 본 지갑 vs 발행 토큰

| 구분 | TronScan 표시 |
|------|---------------|
| **본 지갑 주소** | 문제 안 뜸 (지갑 검색 시) |
| **발행한 토큰 컨트랙트** | "사기/검증 안 됨" 뜸 |

→ **지갑이 문제 아님. 토큰 이름·심볼이 Tether/USDT와 비슷해서 토큰 페이지만 사기로 뜸.**  
→ 지갑 새로 만들어도, 같은 이름 쓰면 토큰 또 사기 뜸. **이름·심볼을 완전히 다르게** 해야 함.

### 3. 메인 지갑 주소 공개 태그 검증 ⭐ (계약 전 필수)

**배포에 쓸 지갑**을 TronScan에 "이 프로젝트 소유"로 등록(불러달라고) 요청하는 거다.

- [ ] https://tronscan.org/#/tools/verify-sign 접속
- [ ] **Sign Message** 탭 → TronLink 연결 → 메시지 `I confirm this address belongs to Link USD.` → Sign → **서명 해시** 복사
- [ ] **Verify Signature** 탭 → Address, Message, Signature 입력 → **Verify** → ✅ Verification passed 확인
- [ ] https://tronscan.org/#/tools/contactUs 접속 → **태그 제출** 선택 → 위에서 복사한 걸 붙여넣기 → 제출

**기입란 상세:** `지갑_주소_검증_기입란.md` (어디에 뭘 적는지 전부)

### 9. TronScan 소스 코드 검증 ⭐ (빼먹지 말 것)

**"계약 코드가 확인되지 않았습니다" 뜨면 이거 안 한 것.**

- [ ] https://tronscan.org/#/contracts/verify 접속
- [ ] **Contract Address**: 배포한 컨트랙트 주소 입력
- [ ] **Main Contract**: LinkUSD (재배포) 또는 TetherUSD (현재 메인넷)
- [ ] **Optimization**: No | **Runs**: 0
- [ ] **Solidity Compiler Version**: 0.8.x (배포 시 사용한 버전)
- [ ] **License**: None 또는 MIT
- [ ] **VM Version**: default
- [ ] **계약 파일 업로드** → "계약서 파일을 업로드하세요" 클릭 → LinkUSD_TRON.sol (LinkUSD) 또는 TetherUSD.sol (TetherUSD) 업로드
- [ ] **Verify And Publish** 클릭

**기입란 상세:** `소스_검증_기입란.md` | **실패 시:** `Verify_실패_해결.md`
